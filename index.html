About

Note from 2024: the text below was written when I had use for this page. I no longer use it. It made sense while my velut project was based in Excel. Now, the project is based in Json data, with scripts that read Json, write Json, and upload Json to the velut database. The Inflector is the script that supersedes this conjugations page.

One benefit of the Inflector is that, because it’s reading Json, extra fields can be added to the input Json to get custom behaviour beyond what’s implied by the principal parts of a verb. With this conjugations page, you might specify that a verb is orior orīrī ortum, but there’s no way of saying the verb is intransitive, has future active participles like oritūrus, or has some other unexpected forms (eg oritur instead of orītur). The Inflector can handle that, and as a script can be easily run again and again without clicking buttons in a page such as this one.

velut is my Latin vocabulary website. I review all words before adding them to velut. This is made faster by generating forms using this webpage.

If you’re not me (Duncan Ritchie), you’re unlikely to have much use for this page. If you’re interested in Latin vocabulary though, you should definitely take a look at velut!

Note on the word “lemma”

The lemma of a word is the form given as the headword in dictionaries; for verbs it’s usually the 1st-person singular present active indicative.

How to use

Enter the three or four principal parts of Latin verbs into the first input box. These can be separated by spaces or commas. Sets of principal parts should be separated by semicolons or line-breaks. (For example: “amō amāre amāvī amātum; doceō docēre docuī doctum; loquor loquī locūtum”.) I provide some sample words that you can load if you want.

Then, click the “Start choosing conjugations” button. A table will appear listing every word you put in, alongside a conjugation — such as “1st, non-deponent” — that has been automatically determined for each word.

This page determines the conjugation based on the final letters of the infinitive, so is frequently wrong. You can, however, edit the conjugation, by deleting the incorrect text and choosing a different conjugation from the list that appears.

Once you are happy with the table of lemmata and conjugations, click “Conjugate”. A list of all the forms will appear in the box underneath. Forms that are stressed on the ending may appear if the “Include stressed endings” tickbox is ticked. The lemma that each form came from may appear beside each lemma (separated by a tab), depending on whether the “Output lemma” tickbox is ticked.

Finally, you can click the “Copy to clipboard” button.

Caveats

Please note: you need to look over all the conjugated forms that have been generated, because this webpage is quite rudimentary and not every inflection makes sense for every lemma.

Also, this page assumes you’re capable of entering letters with macra (the diacritics on ĀĒĪŌŪȲāēīōūȳ) into the first input box.

If the input forms do not end with the letters this page expects, the output forms will probably be wrong.

Some Latin verbs are more irregular in their endings, so this page won’t handle them correctly.

You should also be aware that this page may generate forms that are not attested for the verb: for instance, this page generates passive forms for all non-deponent verbs, but some verbs don’t exist in the passive.

What I use this for

This webpage is convenient for me because much of the data for velut (my Latin dictionary) is generated by and stored in a large Excel file. For each word, I need to generate several values based off the word and the lemma (or lemmata) that the word belongs to. A sheet in my Excel file generates the values I need, based off whatever text is pasted into two of the columns (the “word” and “lemmata” columns).

Therefore, I can paste the output from this webpage into my Excel file, check everything is correct (it probably isn’t), and that’s how I generate everything velut needs for each word.

The de-Excellation of velut

The Excel file that stores, generates, and manages all the data for velut is very large and complex. This is a problem for several reasons — loading and saving the file takes a long time, occasionally it crashes and I lose work, I can’t choose to make some data or some functionality publicly visibly without making it all publicly visible…

That’s why I am in the process of reducing velut’s reliance on Excel, by re-creating all the functionality in the velut Excel file as webpages or websites and by exporting all the data to MongoDB databases or Json files to be used by the webpages/sites.

This webpage is a part of that, since it replaces an Excel sheet that was generating conjugated forms.

I had a similar Excel sheet that generated declined forms of nouns, adjectives, and adverbs. I have replaced it with a webpage similar to this one, but for declensions.

As more and more of velut gets de-Excellated, further changes will be needed to this page. For now, it’s okay.
